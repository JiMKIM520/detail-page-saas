# Task 1: 상세페이지 스크립트 작성

## 1.1 페르소나

```
당신은 국내 Top 5 이커머스 상세페이지 기획 에이전시의 시니어 기획자입니다.

[경력]
- 이커머스 상세페이지 기획 10년+
- 식품/뷰티/전자제품/패션/생활용품/건강기능식품/반려동물 전 카테고리 실전 경험
- 스마트스토어/쿠팡/11번가/G마켓/카카오쇼핑/오늘의집/무신사/위메프/SSG 플랫폼별 규정 정통
- 각 카테고리별 법적 규정(식약처 표시광고법, 화장품법, 전기용품안전관리법 등) 숙지

[작업 원칙]
- 감정과 정보의 비율을 카테고리에 맞게 조절 (식품 70:30 ~ 전자제품 30:70)
- 추상적 표현 금지 → 모든 혜택을 구체적 수치로 정량화
- 법적 금지 표현을 절대 사용하지 않되, 동일한 효과를 내는 허용 표현으로 대체
- 각 섹션의 첫 문장이 해당 섹션의 핵심 셀링 포인트
```

## 1.3 기술 스펙

| 항목 | 값 |
|------|-----|
| 모델 | `claude-sonnet-4-20250514` |
| Max tokens | 4,096 |
| Temperature | 0.7 (창의성과 일관성 균형) |
| 입력 | 텍스트 + 이미지 (상세페이지 스크린샷, 카탈로그 분석) |
| 출력 | JSON (structured output) |
| 비용 | ~$0.03/건 (입력 ~2K + 출력 ~4K tokens) |

## 1.4 제한사항

| 제한 | 대응 |
|------|------|
| 외부 URL 직접 크롤링 불가 | 관리자가 스크린샷 업로드 → Vision으로 분석 |
| SNS 링크 메타 차단 | 상세페이지 URL만 허용 (기존 결정) |
| JSON 파싱 실패 가능 | try/catch + 재시도 로직. `shooting_requirements` 누락 시 기본값 |
| 법적 표현 100% 보장 불가 | QA Agent로 자동 검증 (compliance_score) |
| 카테고리별 필수 섹션 수 상이 | `requiredSections[]`를 JSON 스키마 validation에 적용 |

## 1.5 퀄리티 향상 방안

1. **Few-shot 예시 주입**: 카테고리별 고품질 스크립트 예시 1~2개를 시스템 프롬프트에 포함
2. **정량화 강제**: "모든 형용사는 수치로 전환하라" 규칙 명시 (design-guide.ts PART 1)
3. **금지 표현 테이블**: 카테고리별 `forbiddenExpressions[]`를 "금지→허용" 쌍으로 제시
4. **감정:정보 비율 명시**: 카테고리별 비율을 프롬프트에 직접 삽입
5. **멀티턴 리파인**: 1차 생성 → 자동 검증(법적/품질) → 수정 지시 → 2차 생성
6. **이미지 분석 활용**: 업로드된 상세페이지 스크린샷 + 카탈로그를 Vision으로 분석하여 제품 특장점 자동 추출

## 2.5 퀄리티 향상 방안

1. **NanoBanana 멀티턴 리파인**: 1차 생성 → 결과 확인 → "배경을 좀 더 따뜻하게" 등 수정 요청 (session_id 유지)
2. **카테고리별 프롬프트 분화**: 동일한 "스타일링샷"이라도 식품(시즐컷)≠뷰티(텍스처컷)≠전자제품(스펙샷)
3. **디자인 가이드 PART 2 전량 반영**: 조명 비율, 카메라 설정, 배경, 구도 규칙 모두 프롬프트에 삽입
4. **교차 전략 반영**: 식품×카카오 = 선물 연출, 생활용품×오늘의집 = 룸셋 필수 등
5. **참조 이미지 다중 전달**: 누끼컷 1장(정면) + 1장(45도) + 기존 상세페이지 스크린샷 → 브랜드 톤 일관성
6. **생성 후 자동 검증**: 이미지 크기/비율 확인, 주요 색상 추출하여 브랜드 컬러와 비교

## 3.5 퀄리티 향상 방안

1. **Phase A→B 파이프라인**: 스타일링샷을 먼저 생성하고, 이를 섹션 이미지의 참조로 전달 → 브랜드 톤 일관성
2. **멀티턴 리파인 (NanoBanana 활용)**: 각 섹션 이미지 생성 후 자동 검증 → 문제 시 "텍스트를 더 크게", "배경을 밝게" 등 수정 요청
3. **이전 섹션 참조 전달**: hero 이미지를 benefits 생성 시 참조로 전달 → 전체 페이지 톤 일관성
4. **카테고리별 디자인 패턴 학습**: 
   - 식품: 시즐(감성) → 스토리(산지) → 스펙(영양성분) → 리뷰 → CTA
   - 뷰티: 모델(비주얼) → 임상(데이터) → 텍스처(체험) → 성분 → 루틴 → CTA
   - 전자제품: 히어로(USP) → 기술(인포그래픽) → 시나리오(사용장면) → 비교표 → CTA
5. **Google Stitch MCP 레이아웃 지원**: 섹션 레이아웃 품질을 높이고 싶을 때 `generate_screen_from_text`로 레이아웃 참조 생성
6. **Sharp 후처리**: 생성된 이미지에 정확한 한글 텍스트 오버레이 (Gemini 텍스트 깨짐 대비)
7. **860px 리사이즈 + 최적화**: 생성 이미지를 정확히 860px로 리사이즈. 11번가용 1MB 이하 압축
