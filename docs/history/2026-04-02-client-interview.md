# DetailAI 클라이언트 미팅 인터뷰 가이드

> 일시: 2026-04-02
> 목적: 변경된 요구사항 구체화 및 시스템 수정 범위 확정

---

## 배경

유선 통화로 확인된 방향 변경:
1. 카테고리/플랫폼을 한정적으로 제시
2. 촬영 컷 고정 + 간단 코멘트만
3. 기획자 겸 디자이너 1인이 전체 플로우 관리
4. 플로우: 스크립트 자동생성 → 검토/수정 → 촬영 → AI 상세페이지 제작 → 검토/납품 → 기업 피드백 → 최종수정

---

## 1. 전체 플로우 & 역할 분담

### 기업의 시스템 접근 범위
- [ ] 기업이 시스템에 직접 로그인해서 의뢰서를 작성하나요?
  - 아니면 담당자가 기업 정보를 받아서 대신 입력?
- [ ] 기업이 결과물(스크립트/디자인)을 시스템에서 직접 확인하나요?
  - 별도 채널(이메일, 카톡, 파일 전송)로 전달?
- [ ] 기업 피드백도 시스템 내에서 받나요? 외부 채널?

### 담당자(1인) 작업 방식
- [ ] 현재 스크립트 검수 / 촬영 관리 / 디자인 검수가 별도 화면인데, 하나로 통합된 대시보드가 필요한가요?
  - 또는 탭 분리는 유지하되 계정 하나로 전부 접근 가능하면 충분?
- [ ] 담당자가 스크립트를 직접 수정할 수 있어야 하나요?
  - AI에게 피드백을 줘서 재생성하는 방식? 직접 텍스트 편집?
- [ ] 동시에 여러 프로젝트를 관리할 때 우선순위/정렬 기준이 있나요?

---

## 2. 카테고리 & 플랫폼

### 카테고리
- [ ] 사용할 카테고리 목록은? (현재 시스템: 식품/뷰티/전자제품/패션/생활용품/건강기능식품/반려동물)
  - 이 중 일부만 사용? 새로운 분류?
- [ ] 카테고리는 기업별로 사전에 지정되어 오나요?
  - 한 기업이 여러 카테고리에 해당할 수 있나요?

### 플랫폼
- [ ] 사용할 플랫폼 목록은? (현재: 스마트스토어/쿠팡/11번가/G마켓/SSG/카카오/톡딜/자사몰/기타)
- [ ] 기업별로 플랫폼이 이미 정해져 있나요?
  - 한 기업이 여러 플랫폼 상세페이지를 요청할 수 있나요?
- [ ] 카테고리×플랫폼 고정 조합이 있나요? (예: 식품은 스마트스토어+쿠팡만)

---

## 3. 촬영 프로세스

### 촬영 컷 구성
- [ ] "컷을 정해놓는다"는 것은 모든 기업 동일한 컷? 카테고리별로 다른 컷?
  - 예: 식품 → 메인 1컷, 성분표 1컷, 패키지 1컷
- [ ] 고정 컷 수는 카테고리당 몇 장 정도?
- [ ] 촬영 가이드(각도, 배경, 조명 등)는 누가 제공?

### 촬영 주체 & 프로세스
- [ ] 촬영은 누가 하나요? 자체 촬영팀? 외부 업체? 기업이 직접?
- [ ] 촬영 일정 조율 기능이 시스템에 필요한가요?
- [ ] 촬영본 업로드는 담당자가 하나요? 촬영팀이 직접?

### 코멘트
- [ ] "간단히 코멘트"는 촬영본에 대해 기업이 OK/수정요청 하는 건가요?
  - 사진 각각에 코멘트? 전체 촬영에 한 번에?
- [ ] 코멘트 후 재촬영이 있을 수 있나요?

---

## 4. AI 상세페이지 생성 (디자인)

### 아웃풋 형태
- [ ] AI가 만드는 상세페이지의 최종 결과물 형태는?
  - 이미지(PNG/JPG)? HTML? PSD? 플랫폼 업로드용 파일?
- [ ] 각 플랫폼별 이미지 규격 요구사항이 있나요?
  - 예: 스마트스토어 가로 860px, 쿠팡 가로 780px 등

### 디자인 방향
- [ ] 디자인 템플릿/톤은 카테고리별 통일? 기업별 커스텀?
- [ ] 기업 브랜드 요소(로고, 컬러, 폰트)를 반영해야 하나요?
  - 반영한다면 기업에게 어떤 자료를 받나요?
- [ ] 레퍼런스 디자인을 기업이 제공하나요?

### 생성 과정
- [ ] 스크립트 승인 → 촬영 → 디자인 생성이 자동 연결?
  - 각 단계에서 담당자가 "다음" 버튼을 눌러 진행?
- [ ] 디자인이 마음에 안 들면 재생성? 부분 수정? 완전히 새로?

---

## 5. 기업 피드백 & 최종 수정

### 피드백 방식
- [ ] 기업에게 결과물을 어떻게 보여주나요?
  - 시스템 내 전용 페이지 (링크 공유)?
  - 이미지 파일 직접 전달 (이메일/메신저)?
- [ ] 기업 피드백 형태는?
  - 자유 텍스트 코멘트?
  - 체크리스트 (구성/컬러/문구 등)?
  - 단순 승인/반려?

### 수정 프로세스
- [ ] 수정 요청 시 AI가 자동 수정? 담당자가 수동 수정?
- [ ] 수정 횟수 제한이 있나요? (예: 최대 2회)
- [ ] "마무리"의 기준은? 기업이 최종 확인? 담당자 판단?

## 6. 200개 기업 운영

### 기업 등록
- [ ] 200개 기업 정보는 어떻게 입력?
  - 엑셀/CSV 일괄 업로드? 하나씩 등록?
- [ ] 기업 정보에 포함되는 항목은?
  - 상호명, 사업자번호, 담당자 연락처, 카테고리, 플랫폼, 제품 정보 등
- [ ] 기업 정보가 사전에 확정되어 있나요? 점진적으로 추가?

### 사용 제한
- [ ] 기업당 1회 제한 유지인가요?
  - "1회"의 단위는? 1개 상세페이지? 1개 제품?
  - 여러 제품을 파는 기업은 제품별로 별도 의뢰?
- [ ] 동시 진행 프로젝트 수 예상은? (200개 한 번에? 주 10건씩?)

- [ ] 전체 서비스 오픈 목표 시점은?
- [ ] 파일럿 먼저 진행? 몇 개 기업으로?
- [ ] 가장 먼저 완성되어야 하는 기능은?
  - 스크립트 자동 생성?
  - 디자인 자동 생성?
  - 전체 플로우 관리 대시보드?

### 수정 범위 판단 기준

- **기업이 시스템 직접 사용 → 현재 client 역할 유지, 피드백 루프 추가**
- **기업은 시스템 미사용 → client 페이지 제거, admin 전용으로 전환**
- **1인 통합 대시보드 필요 → UI 구조 재설계 (L 규모)**
- **탭 분리 유지 + 권한 통합 → 권한 설정만 변경 (S 규모)**
