# DetailAI 템플릿 수집·제작·조합 전략 (2026-06-04 리서치 종합)

> 6개 에이전트 병렬 리서치(소싱채널/수집기술/변환방법/법적이슈/품질기준) + 종합. 핀터레스트급(예: 두브로 시그니처 그래놀라) 품질이 목표.

## 0. 핵심 결론 (3줄)
- **수집**: 타사 페이지를 긁는 게 아니라, 레퍼런스에서 **패턴만 추출 → 자체 HTML 슬롯 템플릿 재구현**(저작권상 안전 + 품질 확보).
- **조합**: 섹션을 **공유 디자인 토큰 위에** 만들면, 에이전트가 슬롯 채우기·섹션 조합으로 **품질 페이지 양산 가능**.
- **검증된 스펙**: 리서치가 도출한 "식품 7섹션 슬롯" = 두브로 그래놀라 구조와 일치 → 첫 템플릿 스펙 확보.

## 1. 수집 채널 (카테고리 1순위)
| 카테고리 | 1순위 | 2·3순위 |
|---|---|---|
| 식품 | **스톰데이즈**(stormdayz.com) + **와디즈 펀딩 2000%+** | 핀터레스트 "식품 상세페이지", 크몽 |
| 뷰티 | **슈터**(shooterofficial.co.kr) + **Behance "화장품상세페이지"** | 노트폴리오, 크몽 |
| 전자 | **마상제**(masangze.com, 국내 희소) + Behance "제품디자인" | 글로벌(애플·삼성) 역설계 |

- **핀터레스트는 "발견용"**(로그인벽·썸네일·봇차단) → 수동 보드 큐레이션 → 출처(노트폴리오/Behance) 따라가 고화질 확보.
- **와디즈 펀딩 성공작**이 HOW/WHEN TO EAT·성분 다이어그램·CHECK POINT 등 고급 섹션 패턴이 가장 농축됨.

## 2. 수집·정리 기술
- 캡처: 수동=**GoFullPage**(크롬확장), 배치=**Playwright** `full_page=True + networkidle + 끝까지 스크롤`.
- 중복제거: **imagededup(pHash)** 자동.
- 라이브러리: **Notion DB(인덱스) + 로컬 폴더(파일)**. 메타 6종 — Category/SectionType/Style/Palette/Source/QualityScore.

## 3. 레퍼런스 → 슬롯 템플릿 변환
- **단기(0~3개월)**: 사람 재구축 + **Vision-LLM 초안(~70% 재현, 사람 보정 필수)** 하이브리드. 템플릿 1개 ≈ 8~10시간.
- **중장기(3개월+)**: **섹션 컴포넌트 라이브러리**(15~20섹션) → 조합 엔진으로 무제한 양산. Gemini 이미지 슬롯과 가장 깔끔히 결합. ← 경쟁우위 핵심.
- 미리캔버스/망고보드는 자체 에디터 포맷이라 **코드 파이프라인 연결 불가**(참고만).

## 4. 조합 가능성 (에이전트가 조립?)
가능. 단 **공유 디자인 토큰**(팔레트·타입스케일·여백)이 전제. 모든 섹션이 같은 토큰을 쓰면 에이전트가 조합해도 톤 일관(프랑켄슈타인 방지).
- 에이전트 역할: 카테고리 레시피대로 **섹션 선택·순서 + 슬롯 채우기(카피 Claude / 이미지 Gemini+제품사진) + 브랜드컬러 토큰 주입**.
- 디자인 발명은 안 시킴(토큰·컴포넌트에 미리 박음).
- 품질보증: 렌더→**Vision-LLM 루브릭 채점**→재시도/디자이너 마감(생성↔평가 루프).

### 카테고리별 슬롯 구조 (= 템플릿 설계도)
- **식품(7)**: HERO · INGREDIENT_DIAGRAM · HOW_TO_EAT · WHEN_TO_EAT · CHECK_POINT · TRUST(HACCP/리뷰) · LINEUP  ← 두브로 그래놀라와 동일
- **뷰티(7)**: HERO · TEXTURE · INGREDIENT_MAP · HOW_TO_USE · BEFORE_AFTER · TRUST · LINEUP
- **전자(6)**: HERO · SPEC_INFOGRAPHIC · FEATURE_DETAIL · SETUP_GUIDE · COMPARISON · TRUST

## 5. 품질 큐레이션 루브릭 (수집물 필터)
12항목 × 2점 = 24점, **18점 이상 통과**. 단 **사진 무드 일관성·브랜드 컬러 시스템은 0점이면 자동 탈락**(보완 불가).
- AI-평범 경보(2개↑ 탈락): 기본폰트만 / 그라데이션블러+중앙정렬+범용CTA / 흰·회·검 배경만 교차 / 누끼 클리핑만 / 구분이 hr·동일여백만.
- "진짜 좋은 것"의 신호: ①브랜드 전용 컬러 시스템 ②실제 촬영의 일관된 무드 ③섹션 간 설득 흐름.

## 6. 법적 가드레일
- **아이디어(섹션구조·정보위계·"HOW TO EAT" 명칭)는 차용 가능 / 표현(사진·카피·로고)은 복제 금지**(형사처벌 판례 있음).
- 다수(3개+) 레퍼런스 패턴 조합 → 오리지널 구성 → 자체/Gemini 이미지 + 사업자 제공 사진 = 가장 안전.
- 폰트는 OFL(Pretendard·마루부리 등) 고정, 아이콘 Phosphor(MIT).
- 외주 계약: **저작재산권 전부 양도 + 2차 저작물 작성권 + 소스 납품** 3조항 필수.

## 7. 4주 로드맵 / 비용
- Week1 수집100→큐레이션50~60→패턴추출 / Week2 식품3 / Week3 뷰티3 / Week4 전자2.
- **8개 템플릿(식품3·뷰티3·전자2)으로 200개 사업자 ~80% 커버.**
- 비용: 외주 시 템플릿 1개 50~200만(추정). 섹션 라이브러리는 1회 투자 후 한계비용≈서버비.

## 8. 첫 1개 (두브로 그래놀라급) — 3일 경로
D1 식품 레퍼런스 15개 캡처+루브릭 채점→5개 선정 / D2 섹션 크롭→Vision-LLM 초안→레이아웃·한글폰트·슬롯 보정 / D3 모바일·컬러토큰·재채점→사업자데이터+Gemini이미지 주입 렌더.
