상세페이지 자동 제작 플랫폼
현황 보고
사업자가 제품 정보만 입력하면, AI가 기획·카피·스타일링을 거쳐 상세페이지 초안을 만들고, 디자이너가 Figma에서 마감해 납품하는 폐쇄형 제작 SaaS입니다.
보고일 · 2026.06.02대상 · 식품 카테고리 (소금빵 샘플)상태 · MVP 데모
한 줄 요약
상세페이지 1건 제작에 드는 외주 비용·시간을, 사업자 셀프 의뢰 + AI 초안 + 디자이너 마감 구조로 대폭 줄이는 것이 목표입니다. 현재 핵심 흐름과 결과물 품질을 검증한 MVP 단계입니다.
01 — PROGRESS
진행 상황 · 무엇이 완성됐나
아래 항목은 실제 데이터로 동작을 확인한 내용입니다.
동작 확인운영 웹앱 — 4역할 시스템
관리자 작업관리 대시보드(200건 동시 진행 전제 · 검색/단계/담당자/지연 필터 · 배정), 기획자 스크립트 검수, 디자이너 초안 검수·납품, 사업자 전용 뷰(계정당 1건 제한 · 진행상황 · 초안 확인 · 코멘트). 4역할 로그인부터 전 흐름까지 동작 검증.
동작 확인AI 제작 파이프라인
제품 정보 + 사진 → AI 카피·스크립트 → 디자인 기획(스타일 가이드) → 스타일링샷 프롬프트 → 상세페이지 생성. 각 단계를 실제 제품 데이터로 검증.
완성상세페이지 결과물 — 실사용급 템플릿
히어로 · 핵심가치 · 브랜드 스토리 · 단면 컷 · 가격/구성 · 구매 후기 · 상세 스펙 · 먹는 법 · 배송/교환 · CTA로 구성된 프리미엄 식품 상세페이지. AI 스타일링샷 5종(외관·단면·플레이팅) 적용. 소금빵 샘플 완성.
완성Figma 편집 가능 구조
상세페이지를 이미지가 아닌 편집 가능한 HTML/CSS로 제작 → 디자이너가 html.to.design으로 Figma에 가져와 자유롭게 마감. 모든 텍스트가 정확(이미지에 글자를 굽지 않음).
02 — LIMITATIONS
한계점 · 솔직하게
과장 없이, 현재 시점에서 부족하거나 보완이 필요한 부분입니다.
배포
현재 데모는 로컬 환경에서 구동합니다. 클라우드 배포 시 일부 화면 로딩 이슈가 확인되어, 배포 안정화 작업 후 외부 공개 예정입니다.
자동화 범위
상세페이지는 AI 초안 + 디자이너 마감의 반자동 구조입니다(완전 자동 아님). 또한 새 프리미엄 템플릿은 현재 샘플(소금빵)에 적용됐고, 전 프로젝트 자동 적용은 통합 작업이 남아 있습니다.
카테고리
현재 식품 카테고리 템플릿 1종을 실사용급으로 완성했습니다. 뷰티·전자 등 다른 카테고리 템플릿은 확장 예정입니다.
데이터
가격·구매 후기 등 일부는 샘플 데이터입니다. 실운영 시 실제 판매 데이터·리뷰와 연동됩니다.
03 — ROADMAP
개선 계획 · 다음 단계
1~2주
배포 안정화 + 템플릿 파이프라인 통합(새 프리미엄 템플릿을 전 프로젝트에 자동 적용). 식품 외 뷰티·전자 카테고리 템플릿 추가 착수.
3~4주
구매 후기·리뷰 실연동, 사업자 온보딩(최대 200), 상세페이지 품질 자동 검수(가독성·필수요소 체크) 고도화.
중장기
카테고리 × 플랫폼(스마트스토어·쿠팡·11번가) 조합별 템플릿 다양화, 전환율 데이터 기반 디자인 개선 루프 구축.
