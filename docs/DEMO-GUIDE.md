# DetailAI 데모 가이드

> 실제 사용자처럼 클릭하며 진행하는 데모 시나리오입니다.
> 운영자(디자이너·관리자)와 사업자(신청자) 두 역할로 나뉩니다.

## 접속 정보

| 역할 | 진입 URL | 로그인 |
|------|----------|--------|
| **운영자** | `/admin` | 아이디 `admin` / 비번 `DetailAI!2026` |
| **사업자** | `/login` | 이메일 `demo@detailai.app` / 사업자등록번호 `123-45-67890` |

- 프로덕션: https://detail-page-saas.vercel.app
- 사업자 로그인은 비밀번호 대신 **사업자등록번호**를 입력합니다 (폐쇄형 — 사전 등록된 계정만).

## 데모 데이터 (프로젝트 3종, 사업자 `demo@detailai.app` 소유)

| 제품 | 단계(status) | 데모 용도 |
|------|------|----------|
| 돈덕 순대 | `script_review` | 운영자 **기획 검수 → 승인** 체험 |
| 쌀과밀 소금빵 | `script_approved` | 운영자 **디자인 기획 시작 → 결과** 체험 |
| 황태이야기 | `script_approved` | 추가 기획 데모 |

각 프로젝트에는 실제 누끼컷(배경 제거 제품 사진)이 업로드되어 있습니다.

---

## 시나리오 A — 운영자 (디자이너·관리자)

1. **로그인**: `/admin` → `admin` / `DetailAI!2026` → `/planner`로 이동
2. **기획 검수**: `/planner` 검수 대기 목록에서 **돈덕 순대** 클릭
   - 스크립트(섹션별 카피) 확인 → 우측 검수 패널에서 **승인** 또는 **AI 재생성**
3. **디자인 기획**: **쌀과밀 소금빵**(`/planner/{id}`) 진입 → **"디자인 기획 시작"** 버튼
   - ⏱️ 약 2~3분 소요 (Art Director + Script Writer + 스타일링샷 프롬프트)
   - 완료 시 **디자인 기획 결과**(스타일 가이드: 색상·폰트·레이아웃 + 선택된 템플릿) 표시
4. **촬영 관리**: `/photography` — 누끼/스타일링샷 촬영 요구사항
5. **디자인 검수**: `/designer` — 완성된 상세페이지 디자인 검수

## 시나리오 B — 사업자 (신청자)

1. **로그인**: `/login` → `demo@detailai.app` / `123-45-67890`
2. **내 프로젝트**: `/projects` — 의뢰한 프로젝트 3건과 진행 상태 확인
3. **새 작업 의뢰**: `/intake` — 제품 정보 입력 + 누끼컷 업로드 → 제출
4. **초안 조회·코멘트**: 초안이 준비되면 `/projects/{id}`에서 확인 후 **1회 수정 코멘트** 작성
   - 최종 결과물은 이메일로 발송

---

## 핵심 동작 메모

- **"디자인 기획 시작"** 은 AI 파이프라인을 실제로 실행합니다 (Claude). 약 2~3분 걸리며,
  결과물은 스타일 가이드 + 스크립트 + 스타일링샷 프롬프트 + 선택 템플릿입니다.
- 이미지(스타일링샷)는 자동 생성하지 않습니다 — **프롬프트만 출력**하고, 운영자가 외부 모델(Gemini/GPT)로 직접 추출합니다.
- 폐쇄형 플랫폼이라 공개 가입은 없습니다 (사전 등록 계정만).

## 데모 데이터 재설정

```bash
npx tsx --env-file=.env.local scripts/reset-demo.ts
```

기존 프로젝트를 전부 지우고 위 3종을 초기 상태로 재생성합니다.
