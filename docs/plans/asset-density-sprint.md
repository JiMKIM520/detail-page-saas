# Sprint 후보: 장식 밀도 보강 (아이콘 어휘 + 생성 에셋)

승인: 2026-07-13 (원본 에셋 추출은 라이선스·스타일 통일성 문제로 기각 — 대화 기록 참조)

## 배경
시중 템플릿 대비 시각 밀도 갭의 본질은 레이아웃이 아니라 장식 에셋(커스텀 아이콘·일러스트·텍스처).
원본 .fig 에셋 추출은 ① 재배포 라이선스 위반 소지 ② 프로젝트별 팔레트 변형 시스템과 충돌
③ 스타일 불일치(세트별 스트로크·그리드 상이)로 기각.

## 트랙 A — 아이콘 어휘 확장
- 현재: phosphor 35종 enum (blocks-composer ICON_NAMES). 병목은 소싱이 아니라 허용 어휘.
- 작업: phosphor(MIT, 9,000+종)에서 계열별 어휘 확장(수상·인증·배송·성분 등 아키타입 수요 기준).
  ICON_NAMES 확장 + 컴포저 SYSTEM_PROMPT 어휘 갱신 + 최장일치 수리 사전 동기화.
- 주의: enum이 커지면 오치환 위험도 커짐 — 계열별 서브셋 제시(카탈로그 표본화와 같은 원리) 검토.
- 병행: Reicon 선별 벤더링(라이선스 확인 후) — 기존 백로그 항목.

## 트랙 B — 생성형 장식 에셋
- 파이프라인의 컷 생성 능력을 장식 용도로 확장: 브랜드 팔레트 기반 일러스트·텍스처·패턴 컷.
- 설계도(imageNeeds)에 decorative 니즈 타입 추가 → support 티어(nb2, $0.04)로 생성 → 배경·스트립 슬롯 전용.
- 가드: 장식 컷은 콘텐츠 슬롯 배치 금지(기존 배치 가드에 화이트리스트 추가).
- 비용 영향: 페이지당 +2~3컷 × $0.04 = +$0.1 내외.

## 트랙 C — CSS/SVG 표현력
- 그레인·블롭·패턴 유틸리티를 토큰 체계(형태 언어 5종)에 편입 — 에셋 없이 밀도 상승.

우선순위: A(즉효·저위험) → C → B. 착수 전 A의 오치환 리스크 실측 필요.

## 트랙 D — 밀도 유틸 시스템 이식 (2026-07-15 시안 v2 검증분)
동원 비포/애프터 시안(클라이언트 승인)에서 검증된 코드 장식 층을 렌더러 토큰 계층으로 이식.
전부 CSS/SVG — 형태 언어(5종)·브랜드 팔레트와 연동해 자동 분화.

즉시 이식(시안 v2 실증 완료): 형광 마커·아웃라인 타이포·고스트 타이포·세리프 인용 /
회전 텍스트 스티커·물결 디바이더·손그림 밑줄·점선 화살표·마스킹테이프 / 도트·그리드 배경·
feTurbulence 그레인(다크 섹션, opacity≤.08)·사선 해칭(≤.04) / 오프셋 섀도 칩·오버랩 캡션 카드·
플로팅 제품·radial 스포트라이트(ellipse, 골드≤.22)·mix-blend screen 글로우 / conic 강조 링(페이지당 1회)·
inset 림라이트·그라데이션 보더(border-image)·다층 텍스트 섀도(y≤2px) / 킥커 라벨(—대시)·
세로 사이드 라벨(가독 아닌 텍스처, ≤.3)·원료 스펙 태그(라틴 라벨).

후속(구조 개편·데이터 필요 — 패널 판정):
- 씬 컴포지션(7씬×2000px) 자체가 최우선 — 위 유틸의 캔버스
- 비대칭 칩 그리드·스텝 플로우·그리드 블리드·clip-path 섹션 컷: 변형/컨테이너 구조 재설계 필요
- 레이더 차트·부분 채움 픽토그램·비교 미러: 실측 데이터 확보 선행(무근거 수치 금지)
- 주석 달린 제품 다이어그램: 이미지별 좌표 — 컷 생성과 결합 필요
- backdrop-filter 글래스: 배경 조건 충족 시만

가드레일(패널 공통 경고): 그레인·해칭·글로우는 투명도 상한 고정, conic 링은 페이지당 1회,
다층 섀도 y-offset 2px 이하 — 상한을 토큰으로 강제해 남용(촌스러움) 방지.
