# DetailAI 스태프 인앱 운영 재설계 플랜

작성: 2026-06-19 · 상태: 계획(사용자 확정 대기)

## 1. 배경 / 문제
- 현재 디자인 **생성을 클로드코드(로컬)로 수동** 실행 중 → 기획자/디자이너가 앱에서 직접 운영 불가.
- **Phase0 라이브 검증**(2026-06-19): `/api/designs/generate`를 디자이너 권한으로 실제 트리거 →
  - `intake_submitted → design_generating` 즉시 거부(상태머신, 설계상 정상).
  - `photo_uploaded`로 맞춘 뒤 트리거 → **264초 실행 후 실패**(300초 한도 코앞), 그리고
  - 실패 처리에서 `design_generating → photo_uploaded` **무효 전이 → 2차 에러 → 상태 영구 스턱**(무음에러).
- 결론: 무거운 굽기(chromium)가 Vercel에서 부적합. 화면(UI)은 잘 만들어져 있으나 **동작(생성)이 깨져 운영 불가**. 역할 모델도 실제와 불일치.

## 2. 확정 역할 모델 (사용자 확정)
- **관리자(admin)**: 전 메뉴 접근 / 업체별 담당 디자이너 배정 / 진행 모니터링 / **업체별 촬영(누끼컷) 일정관리**.
- **디자이너(designer)**: 기획자=디자이너 **동일인**. 실명 ~3명 예정. 현재 `planner`/`designer` 분리 역할을 **`designer` 단일로 병합**.

## 3. 확정 워크플로
1. 사업자: 인테이크 제출(제품 실물 발송 안내 포함)
2. 관리자: 담당 디자이너 배정 + **촬영 일정 등록**
3. 디자이너: 스크립트 검수 / 재생성 / 수정
4. (촬영 후) 디자이너: **누끼컷 업로드**
5. 디자이너: **스타일링샷 AI 생성** 검토 / 재생성 / 수정 (모델 Gemini/GPT 미정 → 설정형)
6. 디자이너: **AI 1차 작업물**(블록 컴포저 HTML 초안) 확인/수정 → Figma 임포트 리터치
7. 디자이너: **초안 이미지 업로드(사업자 확인용)** → **사업자에게 자동 이메일 알림**
8. 사업자: 1차 피드백(코멘트)
9. 디자이너: 피드백 반영 → **2차 이미지 업로드**(필요 시 알림)
10. 디자이너: **최종 파일 메일 발송** → delivered

## 4. 아키텍처 결정
- **PNG/PDF 서버 굽기(chromium exporter) 제거** → 생성=경량 HTML만 → Vercel 수초, **별도 워커 불필요**.
- **Figma 플로우(B)**: 블록 컴포저 HTML 초안 → 디자이너가 `html.to.design` 플러그인으로 **Figma 임포트(URL, 수동 1-click)** → 리터치 → **결과를 이미지로 업로드**(시스템이 굽지 않음).
- **초안 HTML 서빙**: Supabase Storage는 HTML을 text/plain+nosniff로 강제(렌더X) → **Vercel 라우트가 text/html로 서빙**(토큰 보호 공개 라우트). PoC: `/samples/...` 공개 경로로 검증 완료.
- **스타일링샷 = AI 이미지 생성**(모델 설정형), 디자이너 검토/재생성.
- **고화질 원본 다운로드**(누끼컷/스타일링샷) 디자이너 제공. (intake-files staff_select 정책 기존재)
- **이메일**: `lib/email/send.ts`(Resend, stub 폴백) **기구현·미연결**. `sendDraftReadyEmail`을 "초안 업로드→design_review"에 연결 + 프로덕션 `RESEND_API_KEY` 설정.

## 5. 고쳐야 할 버그
- 상태머신 롤백 버그: 생성 실패 시 `design_generating → photo_uploaded` 무효 → 스턱. **`design_failed` 상태 추가** 또는 전이 허용 + 실패 사유 표시(무음에러 제거).

## 6. 빌드 순서 (스프린트)
- **S1 — 역할 병합 + 디자이너 워크스페이스 골격**: planner→designer 병합(미들웨어 STAFF_PATHS, role 체크 ~12곳, admin-login, 라우팅, 내비). 한 화면에서 할당 업체의 전 단계를 보는 디자이너 워크스페이스.
- **S2 — 생성 경량화 + 초안 서빙 + 롤백 버그**: exporter 굽기 제거(또는 비활성), 컴포저→HTML→토큰 보호 라우트(text/html) 서빙, 상태 롤백 버그 수정. USE_BLOCKS_COMPOSER 기본화 + 신규 블록 사용하도록 컴포저 가중치 튜닝.
- **S3 — 디자이너 8단계 액션 인앱화**: 스크립트검수/재생성, 누끼컷 업로드, 스타일링샷 AI생성·검토·재생성, AI 1차 초안 확인·Figma 임포트 링크/버튼, 초안 이미지 업로드, 피드백 반영, 2차 업로드, 최종 메일 발송. 각 단계 상태 전이 + 실동작 검증.
- **S4 — 관리자 촬영 일정관리**: 업체별 촬영 일정 등록/조회 + 배정·모니터링 보강.
- **S5 — 이메일 자동 알림 + 다운로드 + E2E**: 초안 업로드→자동 알림 연결, 고화질 다운로드, 전 구간 라이브 E2E 검증.

## 7. 미정 / 결정 필요
- 스타일링샷 이미지 생성 모델: **Gemini vs GPT** (사용자 검토 중).
- Figma 임포트 충실도: 사용자 임포트 테스트 결과 대기(샘플 `/samples/figma-import-test.html`).

## 8. 검증 기준
- 스태프가 **앱에서만**(클로드코드 없이) 한 업체를 인테이크→최종 메일까지 통과.
- 생성이 Vercel에서 안정 완주(굽기 제거 후).
- 초안 업로드 시 사업자 자동 메일 수신.
- build green + 라이브 E2E.
