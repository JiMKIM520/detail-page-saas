# 이미지·조립 파이프라인 재설계 — 역할 분리로 품질 자동 확보

> 2026-07-06 승인. 목표: "하나씩 패치"가 아니라 **시스템이 스스로 품질을 확보**하는 구조.
> 원칙: 생성자-평가자 분리(하네스), 프롬프트 규칙은 확률적 — 최종 방어선은 코드/별도 평가자.

## 1. 배경 — 확인된 구조적 구멍 4종

| # | 구멍 | 실사례 근거 |
|---|---|---|
| 1 | **컴포저 1콜 만능주의** — 한 텍스트 호출이 서사·블록 선택·카피·이미지 배치를 전부 결정 | 효능 카피 옆 라벨 확대, 스텝-이미지 불일치 (4사 캡처 리뷰) |
| 2 | **승인 스크립트 단절** — 고객이 승인한 scripts.content(sections)가 블록 경로에 미전달, DB 원문으로 페이지 재발명 | 브로슈어가 스크립트 생성에는 반영되나 초안에 도달 못 함 |
| 3 | **배치 결정자가 이미지를 못 봄** | 의도≠실물 배치 (TEXTURE에 라벨 확대) — 태거·페어링 QA로 부분 보강됨 |
| 4 | **런타임 평가자 부재** — 컴포저 산출이 무검증 납품 | 모든 캡처 결함이 납품 단계까지 통과 |

## 2. 목표 아키텍처

```
[기획] 승인 스크립트(scripts.content) + 스타일가이드 + 스타일링샷 8컷   ← 기존 유지
   ↓
[A. 페이지 플래너]  승인 스크립트 + 이미지 태거 인벤토리(실물 검수) + 블록 카탈로그
   → 청사진: 섹션 순서·블록(variantId)·섹션별 이미지 배정·카피 지침
   ↓
[B. 블록 필러]  청사진을 구속 계약으로 슬롯 채움 — 카피는 스크립트 문장 재사용/다듬기(창작 최소화)
   ↓
[C. 결정적 가드]  스키마 수리·수치/전화 그라운딩·배치 가드·스텝 균일성   ← 기구축 재사용
   ↓
[D. 페이지 평가자]  렌더 결과를 비전 채점(정합·빈공간·가독성·반복) → 미달 섹션 B 반려(≤2회)
                     ← 페어링 QA는 D의 1차 부품으로 흡수
```

- 템플릿(343변형)·스타일링샷 생성은 **변경 없음** (컷 품질은 누끼 재요청 트랙)
- 비용 목표: 페이지당 Claude ≤ $2 (플래너 소형, 카탈로그는 프롬프트 캐싱)

## 3. 스프린트

### Sprint 1 — 구멍 2·3 봉합 + 제출물 배선 (이번 세션)
- **1a 페이지 플래너**(`agents/page-planner.ts`): 승인 스크립트(scripts.content 우선, 없으면 planning/script.json 폴백) + 태거 인벤토리 + 카탈로그(캐싱) → 청사진 JSON. 규칙: 히어로 시작·클로징 종료, 12~18블록, 이미지 예산·방향 적합, 다양성(동일 계열 연속 금지)
- **1b 컴포저 청사진 수용**: blueprint 입력 시 "구속 계약" — 블록·순서·이미지 배정 준수, 카피는 섹션별 스크립트 발췌 다듬기. 기존 가드·페어링 QA 전부 유지
- **1c 제출물 배선**: ① reference_notes → composeProductContext `[참고사항]` ② brand_logo → 이미지 풀(소형 슬롯 전용 + 배치 가드 화이트리스트 hero/closing) ③ reference_design ≤3장 → 아트디렉터 입력 재연결
- 수용 기준: 황태·럽앤 재생성에서 (a) 페이지 섹션 순서가 승인 스크립트 서사와 대응 (b) 무근거·불일치 0 (c) 참고사항 내용이 FAQ/사용법에 등장

### Sprint 2 — 필러 완전 분리 (후속)
- 필러 프롬프트를 청사진 변형의 계약만으로 축소(343→~15) → 비용↓·정확도↑
- 계약 슬라이스: DATA_CONTRACTS 행 단위 파싱(`variant-id { … }` 형식)

### Sprint 3 — 페이지 평가자 (후속)
- 렌더 세그먼트 스크린샷 비전 채점(루브릭: 카피-이미지 정합/빈 공간/가독성/블록 반복/톤 일관)
- 미달 섹션만 필러 반려, 최대 2라운드. 통과 시에만 designs 업로드
- 페어링 QA를 평가자 내부로 통합

## 4. 리스크
- 플래너-필러 2콜 불일치(필러가 청사진 이탈) → 결정적 검증: 산출 spec의 variantId·이미지가 청사진과 다르면 재시도 피드백
