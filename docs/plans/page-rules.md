# 상세페이지 제작 글로벌 룰 v1 (2026-07-13 검토)

배경: 프로덕션 검수에서 기초 오류 5건 재발(빈 원형·와이드 겹침·누끼 과대·좌측 공백·성분표 소형컷).
공통 근원은 개별 변형이 아니라 **전역 룰 부재** — ① 페이지 폭 표준 없음 ② 부분 결손 렌더 규칙이
변형마다 제각각 ③ 이미지 공급이 블록 선택의 종속변수(샷 수 제각각·활용 저조).
이 문서가 제작 룰의 단일 출처. 코드(렌더러·플래너·가드)는 이 문서 기준으로 정렬한다.

## 1. 페이지 캔버스 (전역 — 렌더러가 보장)
- **고정폭 860px, 중앙 정렬** — 변형이 아니라 페이지 스켈레톤이 강제. 커머스 업로드 규격(780~980) 내.
- 검증·시각 감사 뷰포트 = 산출물 폭과 동일(860). 와이드 뷰포트 의존 겹침 원천 차단.

## 2. 페이지 구성 (플래너 검증이 강제)
- 14~20블록. hero 시작, closing 종료. 동일 변형 2회 금지, 동일 아키타입 3연속 금지.
- 표계열(spec/faq/cs/shipping) 이미지 금지 (현행).
- **[신설] 이미지 보유 블록 하한**: 전체 블록의 50% 이상 이미지 슬롯 보유(슬롯 합 ≥ 10).
  청사진 검증에서 미달 시 재시도 — "16블록에 이미지 2장" 유형의 기획 단계 차단.

## 3. 표준 샷 패키지 (신설 — 샷 개수 표준화)
현행: 니즈 수 = 선택 블록의 슬롯 수(6~15 변동) → 제품마다 들쭉날쭉, 활용도 종속.
제안: **카테고리 공통 12컷 표준 패키지** — 블록 선택과 무관하게 공급 고정, 슬롯이 패키지를 소화.
| 용도 | 컷 수 | 티어 |
|---|---|---|
| 히어로 연출(withProduct) | 1 | main |
| 제품 연출 | 2 | main |
| 원료/소재 | 2 | support |
| 텍스처/디테일 | 2 | support |
| 사용 장면 | 2 | main 1 + support 1 |
| 무드/배경 | 1 | support |
| 스펙/비교 보조 | 1 | support |
| 예비(카테고리 갭) | 1 | support |
- 비용: main 4×$0.14 + support 8×$0.04 ≈ **$0.88/건 고정** (현행 변동 평균과 유사).
- 원본 사진은 패키지와 별도 풀(디테일·신뢰 슬롯 우선, hero·closing 금지 — 현행 가드 유지).
- 목표: 12컷 전량 사용, 미회수 0~2 허용(재배치+갤러리 소진).

## 4. 섹션별 이미지 룰 (아키타입 × 허용 소스 × 결손 처리)
| 아키타입 | 이미지 소스 | 결손 시 |
|---|---|---|
| hero | 연출 생성컷만(원본·단독 누끼 금지, 로고는 소형 슬롯만) | 페이지 실패 — 재생성 |
| story·gallery | 무드/라이프컷, 텍스트 스크림 필수 | 텍스트 강등 |
| ingredient | 원료·텍스처. 누끼전용 프레임=누끼만(현행 가드) | 리스트 전부-또는-0 |
| usage·feature 등 스텝/레일형 | 배열 아이템 이미지 **전부-또는-0** (아키타입 불문) | 아이콘/번호 강등 |
| detail·point·stats | 제품·디테일컷. 2컬럼 스플릿은 이미지 없으면 1컬럼 강등 | 1컬럼 강등 |
| compare | 좌우 각 1 또는 0 | 텍스트 표 강등 |
| spec·faq·cs·shipping | 금지 | — |
| closing | 생성 무드컷만(원본 금지 — 현행 가드) | 그라데이션 강등 |
| 공통(누끼) | 프레임 대비 최대 점유 상한 — 과대 확대·잘림 금지 | — |

## 5. 부분 결손 전역 규칙 (신설 — 기초 오류의 공통 근원 봉쇄)
- 어떤 변형도 **빈 이미지 플레이스홀더(.ph 등)를 최종 렌더에 노출하지 않는다** — 이미지가 없으면
  해당 요소를 숨기고 레이아웃을 재구성(noimg-safe 전수화).
- 렌더 후 결정적 검사: 플레이스홀더 노출 감지 시 해당 블록 드롭(가드 순서: 렌더 직전).
- 배열 아이템 균일화 가드를 usage/ingredient → **전 아키타입**으로 확대(이미지 키는
  mediaSlotKeys로 자동 식별 — 'image' 하드코딩 제거).

## 6. 활용·검증 (현행 유지 + 강화)
- 재배치 패스(니즈 홈→노트 매칭→배열 일괄), 오버레이형 주입 금지, 최소 밀도 min(6,후보).
- 게이트 체인: 태거 → 배치 가드 → 페어링 QA → 평가자(반려 재작업) → 시각 감사(폐루프) → 런 리포트.
