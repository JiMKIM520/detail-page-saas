# DetailAI - 상세페이지 자동화 SaaS

## 프로젝트 개요
200명 사업자 대상 폐쇄형 상세페이지 생성 플랫폼.
카테고리(식품/뷰티/전자제품)×플랫폼(스마트스토어/쿠팡)별 맞춤 상세페이지 자동 생성.

## 기술 스택
- Next.js + TypeScript, Supabase, Claude API
- Vercel 배포

## Quality Rubric (평가자 채점 기준)

평가자(code-reviewer/verifier)는 아래 기준으로 각 항목 1-5점 채점.
**3점 미만 항목이 하나라도 있으면 통과 불가.**

| 기준 | 가중치 | 평가 내용 |
|------|--------|----------|
| **기능 정확성** | 35% | 요구사항대로 동작하는가? (로그인, 결과물 확인, 코멘트, 1회 제한 등) |
| **UI/UX 품질** | 25% | 일관된 디자인 시스템, 반응형, 접근성. 템플릿 느낌 금지. |
| **보안** | 20% | 폐쇄형 인증 체계, 이미지 보호(우클릭/드래그 방지), 데이터 격리 |
| **코드 품질** | 20% | 타입 안전성, 에러 처리, 중복 제거, 테스트 가능한 구조 |

### 채점 예시 (평가자 보정용)
- **기능 정확성 5점**: 모든 유저 플로우(로그인→생성→확인→코멘트)가 에러 없이 동작
- **기능 정확성 3점**: 메인 플로우는 동작하나 엣지 케이스(중복 로그인, 빈 입력 등)에서 오류
- **기능 정확성 1점**: 핵심 기능(로그인 또는 생성)이 동작하지 않음
- **UI/UX 5점**: 의도적 디자인 선택이 보임, 색상/타이포/간격 일관성, 모바일 대응
- **UI/UX 3점**: 기능은 되지만 정렬 불일치, 반응형 깨짐
- **보안 5점**: 미인증 접근 완전 차단, 이미지 보호 우회 불가
- **보안 1점**: URL 직접 접근으로 데이터 노출 가능

## 작업 프로세스 (하네스 패턴)

```
사용자 요청 → [규모 판단]
  S/M: 직접 수정 → 테스트 → 완료
  L/XL:
    1. plan 작성 (docs/plans/)
    2. executor: 스프린트별 구현 + git commit
    3. verifier: Quality Rubric 기준 채점
    4. 미달 시 → 2번으로 (구체적 피드백 포함)
    5. 통과 → Notion 작업 로그 기록
```

## 테스트 검증
- `npm run build` 성공 필수
- Playwright 또는 수동: 로그인→프로젝트 생성→결과물 확인→코멘트 플로우
- Supabase RLS 정책으로 데이터 격리 검증
- 이미지 보호: 우클릭/드래그/개발자도구 기본 차단 확인

## 제약 사항
- 폐쇄형 200명 한정 — 공개 가입 기능 절대 추가 금지
- 기업당 1회 제한 (usage_limit=1)
- 카테고리×플랫폼 조합별 프롬프트 분화 필수
- 배포 전 반드시 확인: auto-deploy 비활성화 상태 (Vercel ignoreCommand 설정)
- Notion 작업 로그 기록 필수
